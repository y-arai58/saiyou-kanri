# 採用管理アプリ セットアップ手順（シンプル版）

## 全体の流れ

```
① スプレッドシート作成
② GASをAPIとして設定・デプロイ（社内ドメイン限定）
③ アプリのコードを書き換えて完成
```

---

## ① スプレッドシートを作る

1. Google Drive で新しいスプレッドシートを作成
2. 名前は何でもOK（例: 「採用管理DB」）
3. 共有設定はそのまま（社内アカウントのみアクセス可）

---

## ② GAS を設定・デプロイする

1. スプレッドシートのメニュー → **「拡張機能」→「Apps Script」**
2. デフォルトのコードを全部消す
3. `gas_api_oauth.js` の中身を全部貼り付ける
4. 上部の `ALLOWED_DOMAIN` を自社ドメインに変更して保存

   ```javascript
   const ALLOWED_DOMAIN = "yourcompany.com"; // ← ここを変える
   ```

5. **「デプロイ」→「新しいデプロイ」**
6. 以下の設定にする：
   - 種類: **ウェブアプリ**
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **自分のドメインの全員（yourcompany.com）**
7. 「デプロイ」→ 権限を許可
8. 表示される **ウェブアプリURL** をコピーして控える

   ```
   例: https://script.google.com/macros/s/AKfyc.../exec
   ```

> ⚠️ コードを修正したら「デプロイを管理」→「編集」→「新しいバージョン」で再デプロイが必要

// ③ 担当者

## ④ アプリの公開方法

### Vercel（推奨・無料）
1. `saiyou_kanri_simple.jsx` を `src/App.jsx` に置く
2. `npm create vite@latest` で React プロジェクト作成
3. GitHub にプッシュ → Vercel で連携してデプロイ
4. 発行されたURLを社内で共有

### Claude.ai のArtifact（お試し用）
- jsxの中身をArtifactに貼るだけですぐ動く
- ただしURLを共有することはできないので本運用には不向き

---

## セキュリティについて

- **GASが社内ドメイン以外のリクエストを拒否**するので、URLが漏れても外部からデータは取得・変更できない
- 社内の人間が社内Googleアカウントでブラウザにログインした状態でのみ動作する
- スプレッドシート自体も社内アカウントのみアクセス可なので二重に守られている

---

## よくあるエラー

| エラー | 原因 | 対処 |
|---|---|---|
| 「スプシ接続エラー」 | GAS URLが間違い or デプロイが古い | URLを確認・再デプロイ |
| データが取れない | アクセス権が「自分のみ」になっている | 「自分のドメインの全員」に変更して再デプロイ |
| 社外から動いてしまう | ALLOWED_DOMAINが未設定 | gas_api_oauth.js のドメインを確認 |
